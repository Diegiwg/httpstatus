--- title: "308" subtitle: "Permanent Redirect" date: 2021-03-01T14:23:15-03:00 draft: false category: "3xx Redirection" references: symfony: Response::HTTP_PERMANENTLY_REDIRECT ---

O recurso de destino foi atribuído a um novo URI permanente e quaisquer referências futuras a este recurso devem usar um dos URIs incluídos.

Os clientes com recursos de edição de link devem vincular novamente as referências ao URI de solicitação efetivo a uma ou mais das novas referências enviadas pelo servidor, quando possível.

O servidor DEVE gerar um campo de cabeçalho Location na resposta contendo uma referência URI preferencial para o novo URI permanente. O agente do usuário PODE usar o valor do campo Location para redirecionamento automático. A carga útil de resposta do servidor geralmente contém uma pequena nota de hipertexto com um hiperlink para o(s) novo(s) URI(s).

Uma resposta 308 pode ser armazenada em cache por padrão; ou seja, a menos que indicado de outra forma pela definição do método ou controles de cache explícitos.

Nota: Este código de status é semelhante a 301 Moved Permanently, exceto que não permite alterar o método de solicitação de POST para GET.

Source: RFC738 Section 3
